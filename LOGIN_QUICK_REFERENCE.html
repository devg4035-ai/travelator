🔄 Authentication Flow

User Input → Validation → Authentication → Session Created → Redirect to Dashboard

🔒 Password Requirements

Your password must contain:

  • ✓ At least 8 characters
  • ✓ At least one uppercase letter (A-Z)
  • ✓ At least one lowercase letter (a-z)
  • ✓ At least one number (0-9)
  • ✓ At least one special character (!@#$%^&*)

Example: Travelator@123

🛡️ Security Features

  • Email format validation
  • Password hashing
  • Failed login attempt tracking
  • Account lockout mechanism
  • Session expiration
  • Email uniqueness validation
  • Password confirmation matching
  • Session storage in localStorage

💾 Data Storage

All authentication data is stored locally in your browser:

  • travelator_users - User database
  • travelator_session - Current session
  • login_lockout_[email] - Failed attempt tracking

Note: For production, use a backend database instead of localStorage.

❓ FAQ

Q: Can I use any password for demo login?
A: No, you must use the exact demo credentials provided above. For new accounts, passwords must meet the requirements.
Q: What happens if I clear browser cache?
A: You'll be logged out as all data is stored in localStorage. You'll need to login again.
Q: How long does the session last?
A: Sessions last 1 hour of inactivity. After that, you need to login again.
Q: Can I change the account lockout time?
A: Yes! Edit the lockoutDuration value in js/auth.js (currently 15 minutes).
Q: Is this production-ready?
A: This is a demo with enhanced security. For production, use bcrypt, JWT, and backend authentication.
